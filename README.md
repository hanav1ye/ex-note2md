# note to Markdown（ブラウザ拡張）

note.com の記事を **Markdown テキスト**に変換する Chrome / Edge 向け拡張機能です。画像は note 上の URL を Markdown に含めますが、**画像ファイルのダウンロードは行いません**。

## できること

- note.com の記事ページ、または記事 URL から Markdown を生成
- **コピー**（クリップボード）または **ダウンロード**（ローカルフォルダへ `.md` 保存）
- 保存先プリセットを最大 3 つ登録し、popup から切り替え
- 同じ note ID のファイルが保存先に既にある場合は **変換・保存をスキップ**（黄色の通知で表示）
- note ページ上の記事リンクを **選択して変換**（単体）
- note ページ上の記事リンクを **複数選択して一括ダウンロード**
- タグ候補から最大 5 つを選び、YAML frontmatter に反映
- 変換結果の先頭に Obsidian 向け YAML frontmatter を付与
  - タイトル・出典 URL・note ID・著者・公開日・タグ・変換日時

### Markdown 変換の対応範囲

| 対応 | 内容 |
|------|------|
| 見出し | `h1`〜`h6` |
| 段落・改行 | 段落区切り、`<br>` はハード改行 |
| 太字 | `**text**` |
| リンク | `[label](url)` |
| 画像 | `![alt](url)`（note 上の画像 URL をそのまま利用） |
| キャプション | `figcaption` を画像直下のイタリックで出力 |
| 引用 | `blockquote` を `>` 付きで出力 |
| コード | フェンス付きコードブロック、インラインコード |
| 水平線 | `---` |
| 除去 | note 付属の目次 UI、記事末尾のハッシュタグ行、コードブロックのコピーボタンなど |

画像ファイルのダウンロードは行いません。Markdown 内には note 上の画像 URL 参照（`![alt](url)`）のみを出力します。

## インストール（開発用）

1. Chrome または Edge で `chrome://extensions`（Edge は `edge://extensions`）を開く
2. **デベロッパーモード**を有効化
3. **パッケージ化されていない拡張機能を読み込む** → このフォルダ `extension-notetozenn` を選択

## 使い方

### 基本フロー

1. 拡張アイコンをクリックして popup を開く
2. **変換元** で「現在のタブ」または「URL を指定」を選ぶ
3. 必要なら **タグ** を候補から選択（複数可）
4. **変換後** で「コピー」または「ダウンロード」を選ぶ
5. **変換** を押す

「現在のタブ」を使う場合は、変換したい note 記事（`https://note.com/.../n/...`）を開いた状態にしてください。初回インストール直後は、記事ページを **再読み込み** してから使ってください。

### ダウンロードを使う場合

1. popup の **プリセットを設定**、または拡張機能の **オプション** 画面を開く
2. プリセットの **フォルダを選択** から保存先を指定する
3. ブラウザのダイアログで **対象フォルダへのアクセスを許可** する
4. popup で保存先プリセットを選び、**変換** を実行する

ダウンロード時のファイル名は note ID ベース（例: `n33f8fe791a43.md`）です。同じ ID のファイルが既にフォルダにある場合は上書きせずスキップします。

### 記事リンクの選択

note.com のページ上で:

- **選択する** … 記事リンクを 1 本クリックすると、その URL で変換・コピー／ダウンロードを実行
- **複数選択** … 記事リンクを複数クリックしてから **実行** で一括ダウンロード（ダウンロードモード時のみ）

## データの保存先（ストレージ）

この拡張は次のストレージを使います。いずれも **ユーザーの端末内** に保存され、外部サーバーへ送信されません。

### `chrome.storage.local`（Chrome 拡張ストレージ）

popup の設定やオプションの内容を保持します。

| キー | 内容 |
|------|------|
| `sourceMode` | 変換元（`tab` / `url`） |
| `outputMode` | 変換後（`copy` / `download`） |
| `articleUrl` | 入力した記事 URL |
| `tags` | popup で選択中のタグ |
| `downloadPreset` | 選択中の保存先プリセット ID |
| `presetConfigs` | 各プリセットの表示名・フォルダ設定有無など |
| `presetTagCandidates` | タグ候補一覧 |

### IndexedDB（`noteToMarkdownPresets`）

File System Access API で選択した **保存先フォルダのハンドル** を保持します（オブジェクトストア `directoryHandles`）。

- プリセットごとにフォルダ参照を保存
- 実際の Markdown ファイル本体は、ユーザーが指定したローカルフォルダに書き込まれます

## フォルダへのアクセス許可について

ダウンロード機能は [File System Access API](https://developer.chrome.com/docs/capabilities/web-apis/file-system-access) を利用しています。

- **オプション画面でフォルダを初めて選ぶとき**、ブラウザがフォルダ選択ダイアログを表示します。ここで保存先を選び、アクセスを **許可** してください。
- 許可しないとダウンロードは実行できません（「保存先フォルダが未設定」「権限がありません」などのエラーになります）。
- ブラウザやプロファイルの状態によっては、**ダウンロードのたびに再許可が求められる**場合があります。その場合はオプション画面からフォルダを再選択してください。
- フォルダ設定を **解除** すると、IndexedDB 上のフォルダ参照も削除されます。

## 権限

| 権限 | 用途 |
|------|------|
| `activeTab` | 開いている note 記事タブの DOM 変換 |
| `clipboardWrite` | Markdown のクリップボードコピー |
| `storage` | 上記 `chrome.storage.local` への設定保存 |
| `https://note.com/*` | 記事ページの読み取り、URL 指定時の HTML 取得 |

## 構成

| パス | 役割 |
|------|------|
| `lib/noteToMarkdown.js` | note DOM → Markdown 変換 |
| `content/content.js` | 記事ページでの変換 API、リンク選択・一括処理 |
| `background.js` | ダウンロード、ファイル存在チェック |
| `popup/` | 変換・コピー・ダウンロード UI |
| `options/` | 保存先プリセット・タグ候補の設定 |
| `manifest.json` | Manifest V3 定義 |

## 変換の元

[note2zenn-hanaviye](https://github.com/) の DOM 変換ロジックをベースに、ブラウザ向けへ移植しています。
